# Storing Data on filesystem ## Advantages - Easiest way to store data without many dependencies is on the filesystem. - Very fast access and read/write speed. - Simpler interaction for quick projects.
# Storing Data on filesystem ## Drawbacks - Very easily misplaced/deleted accidentally - No immediately available backup options ( could copy files to Google Drive/One Drive etc ) - Not able to perform optimised deep queries ( Like SQL WHERE clauses with subqueries and relations... blah blah blah...)

Filesystem Paths

# Absolute Path An absolute path is a path from the root Examples - /home/noah/holy_grail - /home/var - /
# Relative Path A relative path is a path from your current working directory ( `pwd` ) - ( from /home/thomas) `.` points to /home/thomas - ( from /home/thomas) `./mammals` points to /home/thomas/mammals - ( from /home/thomas) `..` points to /home
# So windows is pretty wack... - . would refer to the local directory - \ or / would refer to the *root of the drive you are currently on* - C: would refer to C:\Windows - C:\Foo would refer to C:\Foo
# Permissions Read/Write/Execute
# Python open() flags ``` - "r" --> Default allows read - "w" --> allows write access and creates the file if doesn't exist - "x" --> creates the file exclusively, if the file exists, it errors - "a" --> allows for append only (like appending to end of list) - "b" --> opens in binary mode ( more on that later ;) ) - "+" --> read and write ``` There are more, but not so important ¯\_(ツ)_/¯
